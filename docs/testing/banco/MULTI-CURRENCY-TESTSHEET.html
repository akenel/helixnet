What this tests: a shop prices in ONE home currency but can ACCEPT foreign cash at a rate the admin sets. Block 0 is the setup + the courtesy: an admin-only rate editor in Settings, and a read-only "≈ pay in cash" line at checkout. Cards are unaffected (the customer's bank converts).

Two logins: felix (admin — can edit rates) and pam (cashier — view only). Sandbox is the Roma / EUR shop, so we test "Roma accepts CHF." SANDBOX only — no prod machine-green ≠ human-green — your sign-off

0 · Log in admin + cashier

You'll switch between felix (to set rates) and pam (to confirm she can see but not change them).

Open Banco (sandbox) ↗ https://sandbox-banco.lapiazza.app/pos
  1. Sign in as felix (admin). dashboard shows role admin

1 · Home currency field view all · edit admin

The shop's own currency — displayed to everyone, editable by admin only.

Store Settings → Tax ↗
  1. As felix, Settings → Tax tab. The Currency 🔒 dropdown shows EUR and is editable. dropdown enabled, shows EUR (Roma)
  2. Log out, log in as pam (cashier), same screen. Currency is visible but LOCKED, with a 🔒 "admin only" note

2 · Accepted currencies & rates editor admin only

Where the admin sets the foreign cash the shop takes and at what plan-rate. Roma (EUR) will accept CHF.

Store Settings → Tax ↗
  1. As felix, Settings → Tax → scroll to 💱 Accepted currencies & rates. the section is there, with an ➕ Add currency button
  2. Add a row: currency CHF, rate 1.04 (1 CHF = 1.04 EUR). Click Save. saves without error; the row reads "1 CHF = 1.04 EUR"
  3. Reload the page. the CHF @ 1.04 row is still there — it persisted
  4. Log in as pam, same section. she SEES the CHF rate but can't edit it — no ➕ Add, inputs locked

3 · The "pay in cash" line at checkout the payoff

With CHF accepted, the till should tell the cashier what to collect if the customer pays CHF cash.

New Sale ↗
  1. Ring up a couple of items, go to Checkout. a "💱 pay in cash: ≈ CHF …" line appears under the TOTAL
  2. Check the math against the TOTAL. CHF amount = EUR total ÷ 1.04 (e.g. €10.00 → ≈ CHF 9.62)
  3. As felix, add a 2nd currency (e.g. USD 1.10), save, back to checkout. BOTH show: "≈ CHF … · USD …"
  4. The card / cash buttons themselves are unchanged. this line is INFO only — it doesn't change how you take payment

4 · The safety guard never a wrong number

Foreign amounts must only show when the rates actually belong to this shop's home currency.

  1. As felix, delete ALL rows in "Accepted currencies & rates", save. Back to checkout. the "pay in cash" line DISAPPEARS — cash only in EUR, no leftover numbers
  2. Re-add CHF @ 1.04 to restore it (for the record). line comes back

5 · No regression everything else still works

Adding this must not disturb a normal sale.

  1. Complete a normal EUR sale (cash), take change. works exactly as before — totals, VAT, receipt all in EUR
  2. The Subtotal shows a real number (not "NaN"). earlier fix still holds — no CHF/EUR NaN
