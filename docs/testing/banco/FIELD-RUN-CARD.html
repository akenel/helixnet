The run — a real delivery slip → catalogued

  1. Ask Ralph for the last delivery slip / invoice (they get one most days).
  2. Dashboard → 📥 Receiving.
  3. For each line on the slip: 📷 Scan the barcode (or type it) → Add.
    ✅ Known (already in the catalogue) → it drops straight onto the delivery list.
    🆕 Unknown → the New-item box opens and auto-searches the 7,272-item reference:
      🔎 Match shows up → tap it → name + price + PHOTO fill in → set your cost (or use the box helper: “paid CHF __ for __ units”) → 📦 Adopt & add.
      ✍️ No match? Type the name in the search box first. Still nothing → type name + price + cost → ➕ Add.
  4. Work the whole slip → add a Delivery note (the invoice #) → ✅ Receive N items.
  5. Done — new items are on file (with photos), stock bumped, audit logged. That's your catalogue preload.

Ask Ralph (this is what the trip is really for)

  • Granularity: for papers and cigarettes — do you reorder by brand, and do they have different prices? (different price → each its own item · same price + don't care → one generic line)
  • Deliveries: how often do they come, and how many items on a typical slip?
  • Today's gap: when an item isn't in the till, what do you do right now?
  • Cost/margin: do you track what you paid, or just the sell price?
  • The wall: roughly how many distinct items do you stock — dozens, hundreds?
  • Pain: what's the most annoying part of the current system / stock-taking?
