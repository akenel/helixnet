The argument (read this first)

Big-company systems (SAP, ERP, MRP) keep a perpetual inventory — a running count of every unit. Great for a warehouse with a hundred SKUs that never change. Useless for a kiosk or head shop with a thousand products, new stuff weekly, half of it unbarcoded, old stock piling up. There the count is wrong the moment you write it down — and a wrong number is worse than no number.

So Banco does the opposite: zero perpetual inventory. Don't count — learn the shop by watching what sells (always true; it actually happened). Every item is born once the first time you meet it, then known forever. The catalogue builds itself. The question MRP exists to answer — "what do I re-buy?" — we answer from velocity, not a count nobody keeps honest.

How to read it: read the VO lines aloud (big text), one beat at a time. The (dir.) notes are delivery. Say the punch first, explain after. The green LIVE block = switch to the screen and actually enter a product.
