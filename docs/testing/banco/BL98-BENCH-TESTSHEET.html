What this tests: the enrichment bench (BL-98) — the back-office cockpit that lists every unfinished product with its gaps (photo · description · category · cost), batched a shelf at a time, with a done / total counter that replaces the paper binder. New: a gap filter so you can work ONE missing thing at a time — Angel's ask: "most are just cost — let me see only the ones missing a photo."

Login: felix (admin) or any manager — the bench is manager/admin only. SANDBOX only — no prod machine-green ≠ human-green — your sign-off

0 · Open the Bench manager/admin

The cockpit lives as a MODE on the cleanup screen, not a new page.

Open the Bench (sandbox) ↗ /pos/cleanup?mode=bench
  1. Log in as felix, open Cleanup → 🛠️ The Bench tab. the bench loads: a progress card + a list of unfinished products
  2. Read the big counter. shows done / total and "N still to do" (this is the number that replaces the binder)

1 · The gap filter the new bit

A row of chips — All gaps · 📷 Photo · 📝 Description · 🏷️ Category · 💰 Cost — each with a live count.

  1. Under the shelf box, find the "Show only missing:" chip row. 5 chips, each with a number in brackets, e.g. 💰 Cost (417), 📷 Photo (78)
  2. The numbers should reflect reality — on sandbox most are Cost. Cost's count is the biggest; Photo/Description smaller; they add up sensibly
  3. Click 📷 Photo. the list now shows ONLY products missing a photo; the chip highlights; the counter re-reads for photos
  4. Spot-check 3 rows in the photo view. every one is genuinely missing its photo (no stray fully-set items)
  5. Click 💰 Cost, then 📝 Description. the list swaps to just-cost, then just-description each time; counts match
  6. Click All gaps (or the active chip again) to clear. back to the full unfinished list

2 · One shelf, one gap combine

The shelf (category) filter and the gap filter should stack — "photos missing, in Bongs only".

  1. Type a shelf in Shelf (category) (e.g. a category you saw) → Work this shelf. list narrows to that shelf
  2. Now click a gap chip (e.g. 📷 Photo). list = that shelf AND missing a photo; the chip counts reflect the shelf

3 · Fix one — the counter moves the payoff

The whole point: fill a gap inline and watch the number shrink. The counter is a query, so it can't lie.

  1. With 💰 Cost selected, pick a row and fill its cost (the eyeball can estimate) → Save. the row leaves the list; "still to do" drops by 1; the Cost chip count drops by 1
  2. Do the same with a 📷 Photo item (upload/attach a photo) → Save. it leaves the photo list; Photo chip count drops
  3. Refresh the page. the counts stay dropped — it persisted, not just a screen trick

4 · Nothing else broke no regression

The filter is additive — the rest of the bench must still work.

  1. The Sold tab still shows the sold-but-half-baked safety net. unchanged from before
  2. Export / import worklist (Excel) buttons still work (if you use them). no error introduced
  3. Language: switch to DE/IT. the new chips read in that language — no raw keys like "cleanup.gap_photo"

★ Overall verdict

  1. Bench + gap filter good on sandbox? — tick if it does the job.
  2. Ship BL-98 (with the filter) up the ladder to staging → prod? — tick to green-light the gated deploy.
