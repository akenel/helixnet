🖥️ Kiosk Terminal Setup

Turn the €40 laptop into a locked-down kiosk that powers on straight into Banco. XFCE / Debian-family. 2026-07-15
What we're building: a dedicated kiosk user with no admin power, that the machine auto-logs into on boot and launches Chromium fullscreen at the Banco kiosk — camera on, no address bar, no way to wander into your files. Your angel and root accounts are untouched.

How to run it: open a Terminal on the kiosk laptop, logged in as angel. Every command uses sudo (it'll ask your password). Do the steps in order. After each ⓘ fork, tell Tigs what you saw and he'll confirm your branch before you continue.

0 Tell us the machine

Confirm the distro + which browser is already there. Paste the output to Tigs.

cat /etc/os-release | head -2
command -v chromium chromium-browser google-chrome firefox
ls /etc/lightdm/ 2>/dev/null && echo "LightDM: yes"
ⓘ Fork: note whether it says chromium or chromium-browser, and whether LightDM is present. That decides two commands below.

1 Create the kiosk account (no admin power)

sudo adduser kiosk

It'll prompt for a password (give it its OWN — not your root/angel one) and a few blank fields (just press Enter). This creates /home/kiosk.

Do NOT add kiosk to sudo. Verify it has no admin rights: 
groups kiosk
The word sudo must not appear. If it does, remove it: sudo deluser kiosk sudo.

2 Install Chromium

sudo apt update
sudo apt install -y chromium
ⓘ If that says "Unable to locate package chromium", use this instead:
sudo apt install -y chromium-browser

In kiosk mode there's no address bar, so the guest never sees the brand — Chromium is the same engine as Chrome, camera works the same, and it's one line to install.

3 Auto-login the kiosk on boot (LightDM)

sudo mkdir -p /etc/lightdm/lightdm.conf.d
sudo tee /etc/lightdm/lightdm.conf.d/50-kiosk.conf >/dev/null <<'EOF'
[Seat:*]
autologin-user=kiosk
autologin-user-timeout=0
EOF
sudo groupadd -f autologin
sudo gpasswd -a kiosk autologin

This makes the machine boot straight into the kiosk session — no login screen. (The last two lines satisfy distros that require an autologin group; harmless otherwise.)

4 Auto-launch Banco fullscreen (with camera on)

ⓘ Use chromium or chromium-browser below — whichever Step 2 installed.
sudo mkdir -p /home/kiosk/.config/autostart
sudo tee /home/kiosk/.config/autostart/banco-kiosk.desktop >/dev/null <<'EOF'
[Desktop Entry]
Type=Application
Name=Banco Kiosk
Exec=chromium --kiosk --noerrdialogs --disable-infobars --disable-session-crashed-bubble --use-fake-ui-for-media-stream https://banco.lapiazza.app/pos/kiosk
X-GNOME-Autostart-enabled=true
EOF
sudo chown -R kiosk:kiosk /home/kiosk/.config
The flags, in plain English: --kiosk = fullscreen, no browser UI · --use-fake-ui-for-media-stream = auto-grants the camera so guests never see a permission popup · the rest suppress error/crash bubbles so it always boots clean.
Edit the word chromium to chromium-browser in that file if that's what you installed: sudo nano /home/kiosk/.config/autostart/banco-kiosk.desktop

5 Stop the screen from sleeping

Easiest, click-for-click (do this once logged in as kiosk, or from angel's XFCE): Settings → Power Manager → Display → set "Blank after" and "Put to sleep after" to Never, and untick "Handle display power management". A kiosk should never go black.

5b Wi-Fi must be SYSTEM-owned (not your login's)

The trap: wifi set up under your admin account is saved "me only" + the password sits in your login keyring — so it connects for you but the auto-login kiosk account boots with NO network. Fix it as root so the connection + password belong to the machine:

su -
nmcli connection show                      # find the wifi/hotspot NAME
nmcli connection modify "NAME" connection.permissions "" connection.autoconnect yes 802-11-wireless-security.psk-flags 0
nmcli connection modify "NAME" 802-11-wireless-security.psk "YOUR_WIFI_PASSWORD"
nmcli connection up "NAME"
permissions "" = all users · psk-flags 0 = password stored on the MACHINE, not a login keyring · autoconnect yes = grabbed at boot. (GUI equivalent: tick "All users may connect to this network" on the wifi.)

6 Reboot & test

sudo reboot
🔑 Your escape hatch — read this BEFORE you reboot. In --kiosk mode there's no window to close. To get back to a normal desktop for maintenance: press Ctrl + Alt + F3 (a text login appears) → log in as angel → you're in. To fully undo the kiosk, delete the two files you made: sudo rm /etc/lightdm/lightdm.conf.d/50-kiosk.conf and sudo rm /home/kiosk/.config/autostart/banco-kiosk.desktop, then sudo reboot. You can never lock yourself out — angel and root are always there behind Ctrl+Alt+F3.
What this does and doesn't cover. This gives you the essential security — a guest account with no admin power, no file access, boots into one app. A hardened public kiosk would also disable keyboard escapes (Alt-Tab, virtual-terminal switching) so a clever guest can't break out — that's a later pass. For a shop counter with staff nearby, kiosk-mode Chromium on a powerless account is the 90% that matters. Ship this; harden the last 10% when you're not demoing tomorrow.
