▚ Banco · Product-Entry WorkflowPHOTO-FIRST. One door: New Product → booth photo → AI → search → link. Barcode is the fallback.

One uniform door for every item — "New Product". The barcode path is unreliable in practice (Artemis has no EANs, scans miss, 420 has gaps), so we lead with the photo: it works for everything and doesn't care if the item has a barcode. The system's job is to recognise the item is already in the 5154 catalog and link it — not duplicate. Zero new scraping; the catalog + 420 DB are already in-house.

Every item enters here

➕ NEW PRODUCTThe operator's mental model is always "a new product going in." Behind it, we find + link the existing catalog entry.

The primary flow · photo-first

1
Photo booth → clean shot. Set up the booth day-one; drop the item in for a consistent, well-lit picture.
A clean image is the single biggest lever on how well the AI identifies it.
▼
2
AI describes it. Vision fills in a description (name / brand / type) — even a rough or partly-guessed one is a good search seed.
▼
3
Search the 5154 catalog with that description → candidate matches, each with its catalog photo.
The AI does "what is this?"; the search finds it in the catalog Felix's webshop already built.
▼
◇ Does a catalog photo MATCH the item in your hand?
✓ YES — match found

Link to the existing product (no duplicate) → set the price. Done in seconds.

If the AI's description was off → shorten / fix it → search again → probably lands.

✗ NO — no match

Fallback A · barcode: now "get into the barcode business" — scan → 420 identifies it → find + confirm the Artemis match → link (see the comparison card below).

Fallback B · genuinely new: not in the catalog at all → create it — the photo + AI description are already captured, so it's half-done.

The four rules — locked Angel · 2026-07-11

① Artemis match
Exact name → auto-accept. Fuzzy → show candidate, operator confirms.
② "Your price" default
Pre-fill from the Artemis price — what Felix sells today. Adjust freely.
③ Field ownership
EAN ← 420 · category / 18+ / VAT / tiers ← Artemis · name ← cleaner.
④ Status states
matched-both · 420-only · manual · not-found.
🔑 Why photo-first wins: the barcode path gets stuck (no EAN / miss / gap); the photo path never does — it works on any item, clean shot in, AI + search out. You trade a hair of speed on easy barcode items for a process that always moves. Anyone can do it: photo → confirm → price. The barcode is there for the few the photo can't place.
