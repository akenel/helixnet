│ Data Model                                                                                                        │
│                                                                                                                   │
│ BacklogItem                                                                                                       │
│                                                                                                                   │
│ backlog_items                                                                                                     │
│   id              UUID PK                                                                                         │
│   item_number     int (unique, auto-increment like BUG-001 -> BL-001)                                             │
│   item_type       enum: dev_task, bug_fix, camper_job, business_ops                                               │
│   status          enum: pending, in_progress, blocked, done, archived                                             │
│   priority        enum: low, medium, high, critical                                                               │
│   title           varchar(200)                                                                                    │
│   description     text                                                                                            │
│   assigned_to     varchar(100) nullable                                                                           │
│   due_date        date nullable                                                                                   │
│   estimated_hours float nullable                                                                                  │
│   blocked_reason  text nullable (why is this blocked)                                                             │
│   tags            varchar(500) nullable (comma-separated: "postcard,isotto,urgent")                               │
│   created_by      varchar(100)                                                                                    │
│   created_at      timestamp                                                                                       │
│   updated_at      timestamp                                                                                       │
│                                                                                                                   │
│ No epic/hierarchy in v1. Keep it flat. Tags handle grouping.                                                      │
│ No FK links to bugs/camper jobs in v1 -- just mention them in description. Simpler.                               │
│                                                                                                                   │
│ BacklogActivity (append-only audit trail)                                                                         │
│                                                                                                                   │
│ backlog_activities                                                                                                │
│   id              UUID PK                                                                                         │
│   item_id         UUID FK -> backlog_items                                                                        │
│   activity_type   enum: status_change, assigned, comment, priority_change                                         │
│   actor           varchar(100)                                                                                    │
│   old_value       varchar(200) nullable                                                                           │
│   new_value       varchar(200) nullable                                                                           │
│   comment         text nullable                                                                                   │
│   created_at      timestamp                                                                                       │
│                                                                                                                   │
│ Same pattern as QABugActivityModel. Proven, works.                                                                │
│                                                                                                                   │
│ BacklogComment (separate from activity for richer comments)                                                       │
│                                                                                                                   │
│ Not in v1. Use the activity trail comment type instead. One less table, one less model. Add later if needed.      │
│                                                                                                                   │
│ API Endpoints                                                                                                     │
│                                                                                                                   │
│ GET    /api/v1/backlog/summary          -> counts by status, type, priority                                       │
│ GET    /api/v1/backlog/items            -> list (filters: type, status, priority, assigned_to)                    │
│ POST   /api/v1/backlog/items            -> create new item                                                        │
│ GET    /api/v1/backlog/items/{id}       -> single item + activities                                               │
│ PUT    /api/v1/backlog/items/{id}       -> update (auto-logs activity for status/assignment/priority changes)     │
│ DELETE /api/v1/backlog/items/{id}       -> soft delete (set status=archived)                                      │
│ GET    /api/v1/backlog/items/{id}/activities -> activity log                                                      │
│                                                                                                                   │
