### Step 2 — Test the ownership transfer on **staging** (10 min)

This proves the REASSIGN approach works before touching prod.

```bash
ssh root@46.62.138.218
docker exec -it postgres psql -U helix_user -d postgres
```

```sql
-- Create the staging counterpart user (mirrors what we did for prod)
CREATE USER lapiazza_staging_app WITH PASSWORD '<paste-NEW-staging-password>';
GRANT CONNECT ON DATABASE borrowhood_staging TO lapiazza_staging_app;
\c borrowhood_staging
GRANT USAGE ON SCHEMA public TO lapiazza_staging_app;
GRANT ALL PRIVILEGES ON ALL TABLES IN SCHEMA public TO lapiazza_staging_app;
GRANT ALL PRIVILEGES ON ALL SEQUENCES IN SCHEMA public TO lapiazza_staging_app;
ALTER DEFAULT PRIVILEGES IN SCHEMA public GRANT ALL ON TABLES TO lapiazza_staging_app;
ALTER DEFAULT PRIVILEGES IN SCHEMA public GRANT ALL ON SEQUENCES TO lapiazza_staging_app;

-- THE KEY STEP: transfer ownership of all bh_* tables to the new user
REASSIGN OWNED BY staging_user TO lapiazza_staging_app;

\q
```

Then update `/opt/helixnet/hetzner/.env.staging`:

```bash

docker compose -f hetzner/docker-compose.uat.yml -f hetzner/docker-compose.staging.yml \
    --env-file hetzner/uat.env up -d --force-recreate borrowhood_staging
```
